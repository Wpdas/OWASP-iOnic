Evitar

Usar concatenação simples numa execução SQL. Ao fazer isso, o resultado de uma verificação WHERE por exemplo, sempre vai retornar true caso o usuário informe uma injeção como "or""=". O usuário mal intencionado pode inclusive, em determinado casos, apagar uma tabela ao passar um comando de DROP TABLE.


Recomendado

Usar recursos seguros para informar valores. Por exemplo: dataBase.executeSql('INSERT INTO tabela VALUES (?,?)', ['valor', 'valor']).
