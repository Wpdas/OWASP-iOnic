Qualidade de Código

Evitar

Não testar códigos do projeto. Isso engloba: Teste Unitário, Integração, Funcional, Regressão... Um código defeituoso pode ter furos e erros que podem comprometer seriamente a segurança e o desempenho da aplicação.


Recomendado

Realizar testes em todos os códigos de todas as camadas de desenvolvimento (backend e frontend). Existem diversos softwares prontos a serem usado em diversas linguagens de desenvolvimento como: Mocha, Jasmine, JUnit, Karma, Protractor, etc. Essas são algumas ferramentas que permitem fazer os testes necessários (Unitário, Integração, Funcional, Regressão...).
